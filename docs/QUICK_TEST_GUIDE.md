# 需求分析功能 - 快速测试指引

## 🎯 快速开始（5分钟测试）

### 1. 访问页面
- 打开浏览器，访问：`http://localhost:5173/plan/requirement`
- 或从首页点击"需求分析"按钮

### 2. 选择向导式分析
- 点击"✨ 向导式分析"卡片

### 3. 快速完成5个问题

**问题1 - 产品描述**：
```
输入：这是一款面向18-25岁女性的高品质智能手表
```

**问题2 - 风格选择**：
```
选择：小红书爆款风格、INS极简
（点击多个风格卡片）
```

**问题3 - 卖点选择**：
```
选择：功能、品质、颜值
（点击多个卖点标签）
```

**问题4-5 - 智能追问**（可能不显示，如果产品描述已包含足够信息）：
```
如果显示追问，按提示回答即可
```

### 4. 完成并查看结果
- 点击"完成"按钮
- 查看需求摘要
- 查看自动生成的规划
- 点击"确认规划"跳转到规划页面

## ✅ 核心功能检查清单

### 向导式流程
- [ ] 步骤指示器显示正确（1/5, 2/5...）
- [ ] 进度条平滑更新
- [ ] 可以点击"上一步"返回
- [ ] 每个问题都有验证（空输入、长度限制等）
- [ ] 风格选择支持多选
- [ ] 卖点选择支持多选
- [ ] 完成时显示摘要

### 文本输入模式（向后兼容）
- [ ] 可以选择"文本输入"模式
- [ ] 输入需求描述后可以分析
- [ ] 分析结果正确显示

### 规划生成
- [ ] 完成需求分析后自动生成规划
- [ ] 规划确认弹窗正确显示
- [ ] 可以查看规划详情
- [ ] 确认后跳转到规划页面

## 🐛 常见问题排查

### 问题：向导不显示
- 检查浏览器控制台是否有错误
- 确认已正确导入所有组件

### 问题：规划生成失败
- 检查API配置是否正确
- 或启用Mock模式进行测试

### 问题：数据未保存
- 检查localStorage是否可用
- 查看浏览器控制台的错误信息
